Grid Explícita

Notas:

  • La unidad 1fr = al espacio proporcional del elemento padre, y es el equivalente a la propiedad auto.
  • El shortcut para mover o definir tamaño de los grid cells :
    grid-area : 1 ( valor de inicio de row) / 1 (valor de inicio de columun) / 1 ( valor de final row) / 1 (valor de final column)
  • Para hacer crecer a un elemento partiendo desde su posicion:
    grid-column: span 3; // grid-row: span 2;
  • Para hacer crecer a un elemento partiendo desde su posicion y moverlos a una posicion especifica
    grid-column: 1 (posicion inicial) / span 3 (indice de crecimiento desde su origen ); grid-row: 1 / span 2;

Grid con Áreas

Notas


Es la solucion mas recomendable al momento de diagramar las grids cell, ya que estas se pueden ordenar de forma escrita ( semantica) con la propiedad grid-template-areas

  • Para declarar el aera de forma semantica se usa la propiedad : 
     
   grid-template-areas:

   "header header"
   "content sidebar"
   "content ."
   "footer footer";

    Se entiende que cada cuadrante de la grid esta representado con una palabra separado de un espacio

  • Cada palabra tiene q corresponder con el mismo numero de columnas y filas previamente definidas en el grid-template-columns/rows
  • Definido el espacio solo queda asignarle esta propiedad a cada uno de los elementos hijos con la misma propiedad ejm: 
       
   .header {
   grid-area: header;
   }  
   .content {
   grid-area: content;
   }

   .sidebarr {
   grid-area: sidebar;
   }
   .footer {
   grid-area: footer;
   }
  • Para dejar un elemento vacio, se utiliza la exprecion "." ( punto) en el espacio semantico correspondiente al momento de declarar la proiedad

Flujo de la Grid

Notas

  • Con la propiedad grid-auto-flow se puede definir la direccion en la que se muestra el flujo de los elementos dentro del contenedor, siendo grid-auto-flow:column para señalar que el flujo es en columnas, y grid-auto-flow:rows para las filas
  • Para deifnir el tamaño de los contenedores fuera de la grid implicita se usa grid-auto-rows/column segun el flujo de los elementos hijo.

Grid Items en Capas (superposición)

Notas

  • Para posicionar un elemento dentro de la grid ya establecida, pero que por cuestiones de espacio no entra, y ya forma parte de la grid explicita, solo basta con ubicarla de forma manual usando como sistema de coordenadas las filas y columnas, con esto podemos lograr efectos de superposición. 
              
            css: 

            .grid-layers {
            display: grid;
            grid-template-columns: repeat(4, 1fr);
            grid-template-rows: repeat(4, 1fr);
            }
            .grid-layers .item:nth-child(1) {
            grid-area: 1/1/3/3;
            }
            .grid-layers .item:nth-child(2) {
            grid-area: 1/3/3/5;
            }
            .grid-layers .item:nth-child(3) {
            grid-area: 3/1/5/3;
            }
            .grid-layers .item:nth-child(4) {
            grid-area: 3/3/5/5;
            }
            .grid-layers .item:nth-child(5) {
            grid-area: 2/2/4/4;
            opacity: 0.8;
            }

Alineamiento de Grid Items

Notas

  • Todos los elementos grid cells sse trataran como elementos ITEM
  • Con la propiedad justify-item se puede alinear los items grid cells, todos, por separado y con respecto a su contenedor.
  • Su valor por defecto es justify-item: start | end | center | stretch el cual hace que el hijo ocupe todo el tamaño del contenedor.
  • Esta propiedad se aplicara segun la orientacion del flujo del contenedor padre, esto depende de la propiedad grid-auto-flow: row | column
  • Para aliniear los elementos en su cross axis se usa la propiedad aling-items: start | end | center | stretch
  • Para alinear un elemento por separado se utiliza la propiedad justify | aling - self : start | end | center | stretch

Tamaños Máximos y Mínimos de Grid Tracks

Notas

  • El valor minmax(valor minimo, valor maximo) permite pasar 2 parametros los cuales simularan como primer parametro, el tamaño minimo del contenedor, y el 2do aprametro el tamaño maximo. Ejm: 
      
    grid-template-columns: repeat(4, minmax(100px, 200px));
  • adicional a esto, se agrean 2 unidades de medida a estos parametros como pueden ser min-content , el cual indica que abarque el tamaño minimo que permita el contenido, y por su contraparte max-content el cual indica el tamaño maximo que permita su contenido.Ejm: 
              
              grid-template-columns: repeat(4, minmax(min-content, max-content));
  • Estos parametros se establecen en la propiedad template al definir ya sean filas o columnas.

Grid con Patrones Repetitivos

Notas

  • para crear patrones repetitivos con la propiedad repeat() se tiene que tomar en cuenta que el primer valor que se le vaya a pasar separado de una coma "," es el numero de veces que dicho patron se va a repetir en la grid, y los elementos seguidos de la coma son el patron a definir, ejm: 
              
            .grid-repeat {
            display: grid;
            grid-template-columns: repeat(2, 20% 15% 15%);
            }
  • Hay que tener en cuenta que los elementos se comportaran como bloques en linea, hasta que el patron se cumpla, posterior a esto el grid seguira su flujo de forma explicita.
